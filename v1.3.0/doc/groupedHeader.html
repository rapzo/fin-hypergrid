(static) mixInTo(grid, options)

Mix in the code necessary to support grouped column headers.

Performs the following mix ins:

  1. Creates a new renderer and adds it to the grid.
  2. Extend the behavior's setHeaders method.
  3. Sets the data model's groupHeaderDelimiter property, which tells the data model to prepend the up and down sort arrows to the last item (actual column header) of the header string rather than the start of the header string (which is the highest-order group label).
Parameters:

grid Hypergrid

Your instantiated grid object.
options object

Additions/overrides for the grid's singleton GroupedHeader instance.
options.CellRenderer function <optional>

Extend from this constructor. If omitted, uses the constructor of the cell renderer being used by the first currently active header cell.

(static) paintHeaderGroups()

The GroupedHeader cell renderer's paint function.

This function is called by the Hypergrid renderer on every grid render, once for each column header from left to right. Should be set on (and only on) all header cells — regardless of whether or not the header is part of a column group. (See groupedHeader.setHeaders which does this for you.)

Implements:
  • paintFunction

(static) setHeaders(headers)

Set the headers and set the header row height.

Convenience function to:

  1. Call the underlying setHeaders method.
  2. Set the header row height based on the maximum group depth.
  3. Set all active header cells to use the GroupHeader cell renderer.

  • If a hash: May contain any headers, keyed by field name, in any order.
